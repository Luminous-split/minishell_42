# MINISHELL

*This project has been created as part
of the 42 curriculum by soemin, ksan*

Description:
The goal of this project is to recreate a simplified version of the shell terminal.
Brief overview of minishell; Parsing, Expansion, Child and Parent Process Management, Redirections, Piping, Built-in Commands, Signal Handling and File Handling are the core components of Minishell.

Instructions:
1) Display a prompt and have a working hisory
2) Ensure that signals ^C and ^\ are working properly using at most one global variable and ^D should exit the shell. 
3)Ensure that single quotes, double quotes, unquoted, $? and $VAR (environment variables) can be expanded, interpretated and parsed.
4)Redirections < , > , << , >> must be implemented
5)pipes (| character) must be implemented.
6) The following built-in commnads should be implemented:
  echo with option -n
  cd with only a relative or absolute path
  pwd with no options
  export with no options
  unset with no options
  env with no options or arguments
  exit with no options
